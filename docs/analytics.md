# ContentOS 隐私与分析政策

ContentOS 使用隐私优先的分析来改进开发者体验 (DX)。

## 数据收集原则

### 1. 默认隐私

- **默认不追踪**: 除非您明确同意，不收集任何数据
- **本地存储**: 同意状态存储在浏览器 localStorage
- **随时撤回**: 可随时关闭数据收集

### 2. 收集的数据

#### 基础追踪 (需要同意)
- 匿名事件计数（功能使用频率）
- 页面访问统计
- 错误发生频率
- **不包含**: 个人身份信息、IP 地址、用户代理字符串

#### 增强追踪 (可选)
- 会话数据（用于 TTHW 测量）
- 用户流程（用于识别卡点）
- **仅当您启用** "增强追踪"

### 3. 我们不收集

- ❌ 个人身份信息 (PII)
- ❌ IP 地址
- ❌ 用户代理字符串
- ❌ 地理位置
- ❌ 您创建的内容数据
- ❌ API 密钥或凭据

## 您的权利

### 控制数据收集

```javascript
// 在浏览器控制台执行

// 完全禁用分析
localStorage.removeItem("analytics_consent");

// 启用基础分析
localStorage.setItem("analytics_consent", "true");

// 启用增强分析
localStorage.setItem("analytics_enhanced", "true");

// 禁用增强分析（保留基础）
localStorage.removeItem("analytics_enhanced");
```

### 数据删除

所有数据存储在您的浏览器本地。清除浏览器数据或使用无痕模式将不会记录任何分析。

## 测量目标

我们使用分析数据来改进以下方面：

### Time to Hello World (TTHW)
测量从访问到第一个 Brief 创建的时间。目标: < 2 分钟。

### Journey Drop-off
识别用户在设置流程中的卡点:
- 环境变量配置
- 数据库初始化
- 第一次 AI 生成

### Feature Adoption
了解哪些功能被使用，哪些需要改进。

### Error Rates
追踪 API 错误频率，优先修复最常见的问题。

## 数据安全

- **本地优先**: 数据优先存储在您的设备
- **匿名上传**: 上传到服务器的数据已经去标识化
- **加密传输**: 所有 HTTPS 通信加密
- **无第三方**: 不出售或共享数据给第三方

## 第三方服务

### 未来可能集成

如果我们添加第三方分析服务（如 Plausible、PostHog），将：

1. **隐私优先**: 选择尊重隐私的服务商
2. **明确通知**: 在更新日志中说明
3. **可选启用**: 需要您明确同意

### 服务商对比

| 服务 | 隐私政策 | 自托管 | 是否考虑 |
|------|----------|--------|----------|
| Plausible | GDPR 合规 | ✅ | ✅ 考虑中 |
| PostHog | GDPR 合规 | ✅ | ✅ 考虑中 |
| Google Analytics | 追踪用户 | ❌ | ❌ 不考虑 |

## GDPR 合规

如果您位于欧盟，适用 GDPR 权利：

- **访问权**: 查看我们收集的数据
- **删除权**: 删除本地存储的数据
- **撤回同意**: 随时禁用分析
- **数据可移植**: 导出您的数据

## 开源承诺

分析代码完全开源。您可以：

1. **审查代码**: 查看我们收集哪些数据
2. **自托管**: 在自己的服务器上运行分析
3. **修改配置**: 自定义数据收集策略
